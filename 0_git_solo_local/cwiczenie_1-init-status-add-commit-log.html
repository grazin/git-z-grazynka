Rozpoczynasz pracę nad swoim bardzo ważnym projektem!

#init #status #add #commit #log #vim

$ git init

Stwórz folder o nazwie repo1 i otwórz git bash w tym nowym folderze. Następnie utwórz nowe repozytorium lokalne za pomocą polecenia init.

$ git status

Żeby sprawdzić stan, użyj polecenia status. Przeczytaj ze zrozumieniem output dla pustego repozytorium.

$ git add plik1.txt

Teraz utwórz nowy pusty plik1.txt w twoim repozytorium i ponownie sprawdź status. Przeczytaj ze zrozumieniem. Następnie dodaj plik do indeksu. Użyj komendy add i sprawdź status ponownie.

$ git commit -m "commit-1"

Zacommituj teraz nowo utworzony plik. Przeczytaj output z commita.

$ git log

Teraz możesz też już sprawdzić log twojego repozytorium.

$ git add .

Utwórz jeszcze jeden plik2.txt, a w pierwszym plik1.txt zmień treść. Sprawdź ponownie status. Dodaj zmiany do indeksu. Tym razem użyj komendy add . czyli "dodaj wszystko". Sprawdź status.

$ git commit

Następnie zacommituj te zmiany, ale nie wpisuj parametru -m.


# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
#
# On branch master
# Changes to be commited:
#       modified:  file1.txt
#
~
~

Nie mam pojęcia jaki edytor jest domyślny w tym wypadku na Macu. Ale na windowsie domyślnie będzie Vim. Git bash i tak poprosi cię o wpisanie i zatwierdzenie tytułu commita. Wejdź w tryb edycji i lub Insert. Wprowadź tytul commit-2 i wyjdź Escape a następnie zakończ Shift+zz lub :wq. Sprawdź status i log.

$ git status
$ git log --graph --oneline
