Rozpoczynasz pracę nad kolejną nową funkcjonalnością. Zgodnie ze sztuką robisz to na nowym branchu.

#branch #merge #no-fast-forward

$ git checkout -b task-2

Zacznij od utworzenia nowego brancha o nazwie task-2, użyj polecenia, które od razu przeniesie cię na nowy branch.

$ git branch

Sprawdź jakie masz teraz u siebie branche.

Zrób zmiany w istniejących już plikach, zacommituj wszystkie zmiany, nazwij je commit-task-2.
$ git checkout master

Wróć na branch master. Następnie zrób utwórz nowy plik plik5.txt i zacommituj zmiany , nazwij je commit-4.

$ git merge task-2

Dopiero teraz zmerdżuj zmiany z brancha task-2.

$ git status
$ git log --graph --oneline

Jaka jest różnica między merdżem task-2 a task-1 z poprzedniego ćwiczenia? Sprawdź status i log. Podejrzyj jak to wygląda w GUI
